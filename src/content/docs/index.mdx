## The Tolaria Loop

The original workflow in these docs is the Tolaria Loop:

1. **Capture**: save thoughts, links, notes, and web resources without organizing immediately.
2. **Clarify**: review the Inbox weekly; delete, archive, or keep.
3. **Connect**: relate surviving notes to Projects, Responsibilities, Topics, People, Procedures, or Evergreen ideas.
4. **Commit**: use Git or AutoGit so the knowledge base has durable history.
5. **Compound**: use views, relationships, search, and AI agents to turn accumulated context into writing, decisions, and shipped work.

This borrows from PARA and Zettelkasten, but Tolaria's native design is more specific:
inbox-first capture, typed notes as lenses, relationships as structure, Git as memory,
and AI as a collaborator.
