Types answer the question: "What kind of note is this useful as?"

## Common Types

- Project
- Responsibility
- Topic
- Person
- Procedure
- Evergreen
- Note

These are lenses for navigation. A Project note can show all project work. A
Person note can collect meetings, links, and follow-ups. A Topic note can gather
source material and evergreen ideas.

## Where Type Lives

The note type is stored in frontmatter:

```yaml
---
type: Project
status: Active
---
```

Tolaria does not infer type from the folder path. Moving a note into another
folder does not change what kind of note it is.

## Type Documents

Types can have their own Markdown files in `type/`:

```text
type/project.md
type/person.md
type/topic.md
```

Type documents can define icons, colors, sort order, list display properties,
templates, and visibility. They are still just notes.

## Avoid Over-Modeling

Create a new type when it gives you a durable way to browse or act. Do not create
types just because a field could exist.
