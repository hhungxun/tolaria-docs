Tolaria is opinionated because durable knowledge work needs durable defaults.

## Files First

Your notes are plain Markdown files. They work with any editor, can be searched
from the terminal, and do not require export to leave Tolaria.

## Git First

Git gives every vault a portable history. You can commit, push, pull, inspect
diffs, and recover earlier versions without depending on Tolaria servers.

## Offline First

Tolaria does not require accounts or cloud access. A vault stays useful on a
plane, in a terminal, or in another editor.

## Standards Based

Markdown, YAML frontmatter, wikilinks, folders, and Git remotes are ordinary
tools. Tolaria adds an app experience on top of them without replacing them.

## Types as Lenses

Types group notes and make navigation easier. They are not schemas that block
you from writing.

## AI First, Not AI Only

AI agents work well when knowledge is stored as readable files. Tolaria supports
agent workflows, but the vault remains valuable without AI.

## Keyboard First

Power users should be able to navigate, create, edit, search, and run commands
without leaving the keyboard.
