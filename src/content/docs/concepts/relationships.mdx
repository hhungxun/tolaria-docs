Relationships turn a set of files into a navigable knowledge graph.

## Wikilinks

Use wikilinks in note bodies or frontmatter:

```md
This project depends on [[research-topic]] and [[person-alex]].
```

```yaml
---
belongs_to:
  - "[[responsibility-writing]]"
related_to:
  - "[[topic-markdown]]"
---
```

## Dynamic Relationship Fields

Tolaria treats any frontmatter field containing wikilinks as a relationship. That
means you can create domain-specific relationships without waiting for app changes.

```yaml
---
reviewed_by:
  - "[[person-sam]]"
blocked_by:
  - "[[project-api-migration]]"
---
```

## Default Relationship Vocabulary

These defaults cover many knowledge work cases:

- `belongs_to`: parent or container.
- `related_to`: lateral connection.
- `has`: child or contained item.

## Backlinks and Neighborhoods

Tolaria can show notes that link to the current note. Neighborhood mode pivots
the note list around one note and groups related notes by outgoing and inverse
relationships.
