The Tolaria Loop is the default operating rhythm for a Tolaria vault.

## Why This Works

Most knowledge systems fail when capture and organization become the same step.
Tolaria separates them. Capture stays fast. Clarification becomes a review habit.
Structure appears where it earns its keep.

## Capture

Create notes quickly. Save links. Drop references into the vault. Use Inbox as a
temporary landing zone rather than asking every new thought to be fully modeled.

## Clarify

Review the Inbox on a cadence. For each note, decide:

- Delete it because it no longer matters.
- Archive it because it may matter later but does not need attention.
- Keep it because it should connect to real work or knowledge.

## Connect

Add relationships only after a note survives clarification. Useful connections
usually answer one of these questions:

- What project does this support?
- What responsibility does this improve?
- What topic does this deepen?
- Who is involved?
- What procedure should this change?
- What idea should compound over time?

## Commit

Git makes the vault safer. Commits document how your thinking changed and make it
possible to recover deleted or rewritten notes. AutoGit can automate routine
checkpoints when you trust the flow.

## Compound

Compounding happens when old context becomes useful again. Saved views surface
working sets. Search finds exact language. Relationships reveal neighborhoods.
AI agents can read the same files and help transform context into plans, drafts,
decisions, or shipped work.
