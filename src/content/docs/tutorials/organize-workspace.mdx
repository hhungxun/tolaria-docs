Tolaria organization starts with useful navigation, not a perfect ontology.

## Use Types as Lenses

Create types for repeated ways of looking at your work:

- Project
- Responsibility
- Topic
- Person
- Procedure
- Evergreen note

A type tells Tolaria how to group and display a note. It does not enforce a
database schema.

## Keep Folders Secondary

Folders are supported, but they should not carry the entire system. Prefer
frontmatter and relationships for meaning. Use folders when the filesystem
location itself matters, such as attachments, exports, or narrow working areas.

## Add Views for Work Sets

Create saved views when you repeatedly ask the same question:

- Active projects.
- Blocked work.
- People to follow up with.
- Notes updated this week.
- Writing ideas with enough source material.

Views are YAML files, so they remain easy to review and edit.

## Leave Room for Mess

The Inbox exists because a living workspace is not always clean. The goal is not
zero unorganized notes. The goal is a reliable review loop.
