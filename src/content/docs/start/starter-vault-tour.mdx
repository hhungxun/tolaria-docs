The starter vault is a small, editable tutorial. It is intentionally a real
Tolaria vault rather than a separate help system.

## Key Notes

- **Get familiar with Tolaria**: a checklist for the onboarding path.
- **Principles**: the product philosophy behind files, Git, offline use, and AI.
- **Sidebar**: how Inbox, All Notes, Archive, Favorites, Views, Types, and Folders work.
- **Note List**: sorting, display properties, local search, Neighborhood mode, and creation.
- **Editor**: WYSIWYG editing, raw mode, H1 filename sync, and wikilinks.
- **Properties Panel**: frontmatter, type-aware controls, and relationships.
- **Bottom Bar**: vault selection, Git state, history, version, feedback, and AI status.
- **AI in Tolaria**: quick prompts, AI panel, and CLI agent status.
- **Git and AutoGit**: manual Git control and optional automation.
- **Command Palette**: keyboard-first actions and semantic command search.

## How to Use It

Treat the starter vault as a sandbox:

1. Open a note from the checklist.
2. Try the feature the note describes.
3. Edit a sample property or relationship.
4. Inspect the raw Markdown to see what changed.
5. Commit or discard changes when you are done.

The vault is local by default, so experiments do not affect the upstream starter
repository.
