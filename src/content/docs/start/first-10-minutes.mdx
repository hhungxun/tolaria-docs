Use this checklist in a fresh Getting Started vault.

## Minute 1: Orient

Look at the four main areas:

- Sidebar: global navigation, Inbox, views, types, folders.
- Note list: the selected slice of the vault.
- Editor: the current note.
- Properties panel or AI panel: metadata, relationships, history, or agent work.

## Minute 2-3: Read the Principles

Open the Principles note. The core idea is simple: files first, Git first,
offline first, standards based, and AI-ready without requiring AI.

## Minute 4-5: Try Navigation

Click a type in the sidebar, then click a note. Use a wikilink in the note body
or a relationship in the Properties panel to move to a related note.

## Minute 6-7: Edit a Note

Create a short note, add a heading, then type `[[` to link another note. Switch
to raw mode if you want to see the exact Markdown.

## Minute 8: Add Structure

Open the Properties panel and add a `type`, `status`, or relationship. Remember:
types are lenses, not rigid schemas.

## Minute 9: Check Git

Look at the bottom bar. If the vault is Git-backed, Tolaria shows changes, history,
commit, push, and sync state there.

## Minute 10: Ask an Agent

If you have a supported CLI agent installed, open quick prompt mode from the
command palette or the AI panel. Ask about a note and pass context with wikilinks.
