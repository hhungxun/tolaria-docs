Saved views are YAML files in `views/`.

## Minimal View

```yaml
name: Start here!
icon: null
color: null
sort: property:onboarding:asc
filters:
  any:
    - field: type
      op: equals
      value: Project
    - field: related_to
      op: contains
      value: Tolaria
```

## Filters

Use `all` when every condition must match. Use `any` when at least one condition
can match. Nested groups allow more expressive filters.

## Sort

Examples:

```yaml
sort: modified:desc
sort: title:asc
sort: property:Priority:asc
```

## Sidebar Order

Views can carry order metadata so they appear predictably in the sidebar.

## Editing

You can edit views through Tolaria, in a text editor, or with an AI agent. Keep
the YAML readable and commit changes when a view becomes part of your workflow.
